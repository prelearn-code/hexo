1. 类型

1.1 数值类型

说明：由于以太坊中需要确定的状态变化，不允许有浮点数的存在，因为浮点数的计算精确度不能确定，占用空间比较小，可以直接赋值。

a . 布尔型（bool）

运算符号：
！：逻辑非。
||：逻辑或。
&&：逻辑与。

b. 整数类型（int uint uint256等）

运算符号：正常的运算符号。

c. 地址类型

  • 普通地址：20字节的值。
  • 可支付地址：多两个转账函数。

d.定长字节数组

属于值类型，数组长度在声明之后不能改变。根据字节数组的长度分为 bytes1, bytes8, bytes32 等类型。定长字节数组最多存储 32 bytes 数据，即bytes32

e. 枚举类型enum

枚举（enum）是 Solidity 中用户定义的数据类型。它主要用于为 uint 分配名称，使程序易于阅读和维护。它与 C 语言 中的 enum 类似，使用名称来代替从 0 开始的 uint

b.映射原理

2.数据存储位置与变量的作用域

2.1数据存储位置

2.1.1 storage

说明：默认数据为存储上链的。

赋值特点：再合约storage赋值给函数本地的storage类型时，为引用方式进行，修改一个的数据内容就会改变另一个数据的内容。

2.1.2 memory

说明：函数里的参数和临时变量一般用memory，存储在内存中，不上链。尤其是如果返回数据类型是变长的情况下，必须加memory修饰，例如：string, bytes, array和自定义结构。

赋值特点：在memory赋值memoey类型时，也是引用赋值，改变一个数值就会修改另一个数据的内容。

2.1.3 calldata

说明：也是不上链，存储在内存中，与memory不同的时不能够进行修改变量。

2.2变量的作用域

2.2.1 状态变量

合约内部定义的变量。

2.2.2 局部变量

函数内部定义的变量。

2.2.3 全局变量

自带的变量类型，address之类的。

2.3 数据的初始值

说明：delete能够使数据变为初始值。

2.3.1数据类型的初始值

  • boolean: false
  • string: ""
  • int: 0
  • uint: 0
  • enum: 枚举中的第一个元素
  • address: 0x0000000000000000000000000000000000000000 (或 address(0))
  • function
    • internal: 空白函数
    • external: 空白函数

2.3.2引用类型的初始值

  • 映射mapping: 所有元素都为其默认值的mapping
  • 结构体struct: 所有成员设为其默认值的结构体
  • 数组array
    • 动态数组: []
    • 静态数组（定长）: 所有成员设为其默认值的静态数组

3.函数

3.1 函数性质

  • parameter types：数据类型 参数名称 （uint number）

  • internal|external|public|private

    • internal：只能从合约内部访问，继承合约可以使用。
    • external：只能从合约外部访问（但内部可以通过 this.f() 来调用，f是函数名）。
    • public：内部和外部均可见。
    • private：仅本合约可见。

  • pure|view|payable

    • pure：类似公式，既不能读取数据，也不能进行修改数据，只能进行计算，返回计算结果。
    • view：只有读取数据的权限，没有进行修改的权限。
    • payable：可支付的合约。

3.2函数返回值

return(a,b,c,d)：可以根据数据进行指定返回。

同时可以进行解构赋值。

4. constant和immutable

  • constant：在声明时必须进行初始化数据，并且后续数据不可改变。
  • immutable：在声明时定义或者在之后进行一次赋值定义，获得初始值之后不可再改变。

8.继承

virtual: 重载函数需要的关键字.
override：重载后的函数需要添加的关键字。

在继承时，对于继承的父类，多重继承需要对于父类进行排序，辈分从高到底，依次为从左到右。

9.抽象合约与接口

抽象合约：存在至少一个函数未被实现，只存在函数的声明，需要在合约前添加abstract.
接口：只有函数声明，没有函数实现，类似于一个合约函数的模板。

  1. 不能包含状态变量
  2. 不能包含构造函数
  3. 不能继承除接口外的其他合约
  4. 所有函数都必须是external且不能有函数体
  5. 继承接口的非抽象合约必须实现接口定义的所有功能

10.异常

10.1 error

用法与事件比较相似。使用自定义错误并结合 revert 语句，可以提供详细的错误信息，并且相比字符串错误消息更加节省 gas。
类似于其它语言中的条件提示。判断进程的运行过程。

10.2 require

如果条件不满足，则中止执行并撤销当前交易，退还剩余的 gas。

10.3 avert

assert 失败会消耗掉所有剩余的 gas，并抛出错误。它通常表示合约中存在不可预期的错误或漏洞。

11.函数重载

Solidity中允许函数进行重载（overloading），即名字相同但输入参数类型不同的函数可以同时存在，他们被视为不同的函数。注意，Solidity不允许修饰器（modifier）重载。与c++的函数重载类似。

12. 库合约与import

12.1库合约

12.1.1库合约特点

  1. 不能存在状态变量
  2. 不能够继承或被继承
  3. 不能接收以太币
  4. 不可以被销毁
    库合约重的函数可见性如果被设置为public或者external，则在调用函数时会触发一次delegatecall。而如果被设置为internal，则不会引起。对于设置为private可见性的函数来说，其仅能在库合约中可见，在其他合约中不可用。

12.1.2常用库合约

  1. Strings：将uint256转换为String
  2. Address：判断某个地址是否为合约地址
  3. Create2：更安全的使用Create2 EVM opcode
  4. Arrays：跟数组相关的库合约
